Lists

We currently have the following mailing lists for our project:

iguana-developers@cern.ch (archive)
The developer list; everybody working on IGUANA is subscribed to this list. If you have any technical questions, problems or enhancement ideas, please use this list to reach the developers. Note that it is better to use the list than to send mails to us individually as that allows all of us to respond and stay aware of the status quo. (List administration)

iguana-interest@cern.ch (archive)
Announcements and general IGUANA information. Low volume. (List administration)

iguana.support@cern.ch
Please use the developer list mentioned above instead.

Mailing list commands

If you have an account at CERN, the easiest way to manage your subscriptions and to get information is via SIMBA. Otherwise use the following commands. Note: remember to send your requests to the list manager, not to the IGUANA lists!

subscribe list-name [address]
Subscribe to the list. If you don't specify an address, the address in the "From" (or Reply-To) header of your message will be used.
unsubscribe list-name [address]
Unsubscribe from the list. If you are sending the mail from an address that is different from your subscription address, you should specify the address in your message.
which [address]
Find out which lists you (or the address given) are subscribed to.
who list-name
Find out who's subscribed to the list.
info list-name
Get the list info blurb.
index list-name
Get the list of files in the list archive.
get list-name file
Retrieve a file from list archive.
lists
Get a catalog of available lists.
help
Get comprehenive help about the list manager service, including lots more details about all the commands listed here.
end
Stop processing commands. Useful if your signature has words that might confuse the list manager.

For example, to add yourself to the iguana-developers list, simply follow this link: majordomo@cern.ch. This will send a message to majordomo with this message body:

subscribe iguana-developers
help

Note that majordomo understand several commands, it is not limited to a single command per message. It will process commands until the end of the message or the command "end". The "help" command will tell you all about the other commands.
