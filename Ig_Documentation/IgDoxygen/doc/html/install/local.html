Running IGUANA

The easiest way to run IGUANA is to setup the IGUANA environment. Here is what you can do:

  • cd /some/home/area
  • scram list IGUANA (optional; shows which releases are available)
  • scram project IGUANA @LAST_VERSION@ (creates a development area)
  • cd @LAST_VERSION@
  • eval `scram runtime -csh` (setup environment, use -sh if you are a bash/sh user)
  • iguana (runs iguana)

You can also go to the IGUANA release area and run iguana from there

  • scram list IGUANA (optional; shows which releases are available)
  • cd IGUANA-Release/@LAST_VERSION@ (IGUANA-Release is the path shown by the above command for @LAST_VERSION@ release)
  • eval `scram runtime -csh` (setup environment, use -sh if you are a bash/sh user)
  • iguana (runs iguana)

Building IGUANA locally

The easiest way to build IGUANA is to use the make-release script as described in the release how-to guide. The key SCRAM build commands available in the top src directory are:

  • scram b release: builds the entire release
  • scram b release-build: builds all the binaries, test programs and the dependency diagrams
  • scram b release-check: runs all release tests
  • scram b release-docs: builds documentation (linux)
  • scram b: builds all binaries (programs and libraries but not the test programs)

Doing development for IGUANA

If you wish to do some development, you should create a new working area with SCRAM like this:

  • project IGUANA
  • scram project IGUANA @LAST_VERSION@ (the last argument is the IGUANA version you wish to use; please refer to the output of scram list IGUANA to see which versions have been installed)
  • cd @LAST_VERSION@/src
  • eval `scram runtime -csh`
  • Check out the code you are going to modify, e.g. one of the examples:
    cvs co Ig_Examples/IgG4Demo
  • scram b (build)
  • iguana (run)

    • For further instructions on how to use IGUANA, please refer to the user guide. For more information on how to develop with IGUANA, please refer to the developer guide.
